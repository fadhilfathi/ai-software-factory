## ¬ß2 6-state lifecycle verification

The headline invariant: **a `running` execution can ONLY reach `completed` via `review`; there is no direct `running ‚Üí completed` edge.**

### Runtime check (service/execution.go:validExecutionTransitions)

```
queued    ‚Üí {assigned, failed}
assigned  ‚Üí {running, failed, queued}
running   ‚Üí {review, failed}                          ‚Üê no completed
review    ‚Üí {completed, failed}                      ‚Üê only path to completed
completed ‚Üí {}                                        ‚Üê terminal
failed    ‚Üí {}                                        ‚Üê terminal
```

The `running ‚Üí completed` edge is **structurally absent** from the state machine map. The `isValidExecutionTransition` function returns `false` for it (and surfaces as `ErrInvalidStateTransition` ‚Üí 409 INVALID_STATE_TRANSITION in the handler).

### Runtime check (service/execution.go:driveWorker)

When `aion.WorkerCompleted` is received from the runtime (line 631‚Äì636), the transition target is `model.ExecutionStatusReview`, **not** `ExecutionStatusCompleted`. The comment block (line 585‚Äì591) spells out the security posture:

```go
// B-001 6-state handoff: worker-complete transitions the
// execution to REVIEW, not COMPLETED. The reviewer action
// (PATCH /v1/executions/:id/review) is the only path into
// COMPLETED. This matches api-spec ¬ßThe Execution Engine.
```

### Runtime check (service/execution.go:CancelExecution)

Operator cancel (line 651‚Äì670) only operates on non-terminal rows. Terminal rows (completed/failed) return `ErrInvalidStateTransition` (line 661‚Äì663), preventing the operator from overwriting a finished audit trail.

### Runtime check (service/execution.go:ReviewExecution)

The reviewer action (line 596‚Äì649) is the **only** path into `completed`. It requires `status=review` (line 619), and the verdict `accepted=true` transitions to `completed` while `accepted=false` transitions to `failed` (with the rejection reason required and stored in `error_message`).

### Test check (execution_state_table_test.go)

- **`TestExecutionStateMachine_TableDriven/Running_ToCompleted_BLOCKED`**: the explicit test of the headline invariant. `UpdateExecutionStatus(ctx, exec.ExecutionID, model.ExecutionStatusCompleted, ...)` from a `running` row must return `ErrInvalidStateTransition`. PASS by construction (the test was authored after c2 + c3, when the runtime check was already in place).
- **`TestExecutionStateMachine_TableDriven/Review_ToCompleted_Accept`**: confirms that the reviewer path works.
- **`TestExecutionHandler_Review_Accept_200`** (handler test): HTTP-level confirmation that the reviewer endpoint is the only wire path to `completed`.
